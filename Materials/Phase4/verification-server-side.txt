3) Server-Side Validation Tests (HTTP Requests)

Test A - Invalid action
- Payload:
  - `{"action":"delete","resourceName":"Room A","resourceDescription":"Valid Description","resourceAvailable":true,"resourcePrice":5,"resourcePriceUnit":"day"}`
- Expected: rejected
- Actual:
  - Status `400`
  - `{"ok":false,"errors":[{"field":"action","msg":"action must be 'create'"}]}`

Test B - Empty resource name
- Payload:
  - `{"action":"create","resourceName":"   ","resourceDescription":"Another valid text","resourceAvailable":true,"resourcePrice":5,"resourcePriceUnit":"day"}`
- Expected: rejected
- Actual:
  - Status `400`
  - `{"ok":false,"errors":[{"field":"resourceName","msg":"resourceName cannot be empty"}]}`

Test C - Invalid availability type
- Payload:
  - `{"action":"create","resourceName":"Room A","resourceDescription":"Valid Description","resourceAvailable":"maybe","resourcePrice":5,"resourcePriceUnit":"day"}`
- Expected: rejected
- Actual:
  - Status `400`
  - `{"ok":false,"errors":[{"field":"resourceAvailable","msg":"resourceAvailable must be boolean"}]}`

Test D - Invalid price unit
- Payload:
  - `{"action":"create","resourceName":"Room A","resourceDescription":"Valid Description","resourceAvailable":true,"resourcePrice":5,"resourcePriceUnit":"year"}`
- Expected: rejected
- Actual:
  - Status `400`
  - `{"ok":false,"errors":[{"field":"resourcePriceUnit","msg":"resourcePriceUnit must be 'hour', 'day', 'week', or 'month'"}]}`

Test E - Malformed JSON
- Payload body:
  - `{bad json}`
- Expected: rejected with meaningful error
- Actual:
  - Status `400`
  - `{"ok":false,"error":"Invalid JSON payload"}`

4) Functional Create Test (Valid Request)

Test F - Valid create
- Payload:
  - `{"action":"create","resourceName":"Verification Room B","resourceDescription":"Valid room for verification run","resourceAvailable":true,"resourcePrice":16,"resourcePriceUnit":"week"}`
- Expected: created with correct values
- Actual:
  - Status `201`
  - `{"ok":true,"data":{"id":7,"name":"Verification Room B","description":"Valid room for verification run","available":true,"price":"16.00","price_unit":"week","created_at":"2026-03-08T21:18:32.814Z"}}`

5) Database Check (After Testing)
- Query used:
  - `SELECT COUNT(*) FROM resources;`
- Result after tests:
  - `COUNT_AFTER = 7`
  - `DELTA = 1`

Interpretation:
- Only the valid request inserted a new row.
- Invalid and malformed requests were rejected and did not create database rows.
